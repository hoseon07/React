## 2. SPA → single page application

- 렌더링 역할을 서버에게 넘기지 않고 브라우저에서 처리하는 방식이다.

### 장점

- 전체적인 트랙픽 감소, 렌더링에서 좋은 효율을 가진다.
- 빠른 화면 이동이 가능하다.
- 모듈화또는 컴포넌트별 개발이 용이하다.

### 단점

- 리소스를 한번에 다운로드 하기 때문에 초기 구동 속도가 느리다.
- 데이터 처리를 클라이언트에서 하는 경우가 많음 → JS를 통해 구현되므로 코드가 외부에 노출됨
